# 노동요 셔플 표시 전용 및 원본 순서 재생 작업 보고서

## 원래 요청

셔플을 켜도 여전히 곡이 순서대로 재생되지 않는 문제를 수정한다. 셔플은 아래 곡 목록의 표시 순서만 랜덤으로 바꾸고, 실제 재생은 셔플 상태와 관계없이 저장된 원본 곡 순서대로 진행한다.

## 다시 확인된 원인

- 직전 수정에서는 “화면에 표시된 셔플 배열 안에서 순차 재생”으로 요청을 해석했다.
- 이 때문에 목록 배열 자체는 고정되어도 원본 곡 순서 기준으로는 여전히 `3 → 1 → 4 → 0 → 2`처럼 랜덤 재생됐다.
- 사용자가 원하는 동작은 화면 표시만 랜덤이고 재생은 항상 원본 인덱스 `0 → 1 → 2 → …`를 따르는 것이었다.

## 완료한 작업

### 재생 경로에서 셔플 완전 제거

- `getWorkMusicAdjacentIndex()`에서 셔플 모드 분기를 제거했다.
- 다음곡은 항상 `(현재 인덱스 + 1) % 곡 수`로 계산한다.
- 이전곡은 항상 `(현재 인덱스 - 1 + 곡 수) % 곡 수`로 계산한다.
- 수동 이전·다음 버튼, Cover Flow 이전·다음 커버, 하단 리모컨 미리보기가 모두 원본 순서를 사용한다.

### 플레이어 재생목록 원본 순서 고정

- `getWorkMusicPlayOrder()`가 셔플 모드를 확인하지 않도록 변경했다.
- 일반 YouTube 플레이어와 이어듣기 플레이어 모두 원본 인덱스 배열만 사용한다.
- 현재 곡이 원본 배열 중간에 있으면 해당 곡부터 `현재 → 다음 → … → 마지막 → 첫 곡` 순서로 순환한다.
- 자동 곡 종료, 이어듣기 대기곡, 실패곡 건너뛰기도 이 원본 순서를 사용한다.

### 셔플 기능의 역할 제한

- 랜덤 배열 생성은 `getWorkMusicDisplayOrder()` 한 곳에만 남겼다.
- `random` 모드 확인은 목록 표시, 버튼 활성 표시, 모드 전환에만 사용된다.
- 목록이 예를 들어 `3 → 1 → 4 → 0 → 2`로 보여도 실제 재생은 `0 → 1 → 2 → 3 → 4`로 진행한다.
- 버튼 명칭은 직전 수정대로 `목록 셔플`을 유지한다.

## 기존 기능에 미치는 영향

- 셔플 모드의 실제 재생 의미만 변경했다.
- 곡 목록의 랜덤 표시와 탭별 셔플 캐시는 유지된다.
- YouTube 플레이어 생성 옵션, 이어듣기 전환 방식, 볼륨, 오류 판정, 감시 타이머는 변경하지 않았다.
- 곡 클릭 재생, 재생·일시정지, 재생바 조절 기능은 유지된다.
- 저장되는 기존 모드 값 `random`과 `sequential`은 그대로 사용한다.

## 변경 파일

- `src/features/workmusic.js`: 모든 재생·이전·다음 경로를 원본 인덱스 순서로 고정
- `index.html`: 직전 수정의 `목록 셔플` 버튼 설명 유지
- `current_task.md`: 이번 재분석 및 수정 내용으로 갱신

## 검증 결과

### 완료

- JavaScript 문법 검사 통과
- 전체 ESLint 경고 및 오류 없이 통과
- 전체 Prettier 검사 통과
- `git diff --check` 통과
- `random` 모드 분기가 목록 표시와 UI 상태에만 남아 있는지 검색 확인
- 일반 모드와 셔플 모드 모두 다음곡이 항상 원본 인덱스 `+1`, 이전곡이 `-1`인지 수치 확인
- 다섯 곡의 모든 시작 인덱스에서 플레이어 순서가 원본 배열을 순환하는지 확인
- 셔플 표시 배열 `[3, 1, 4, 0, 2]`와 무관하게 실제 재생 배열이 `[0, 1, 2, 3, 4]`인지 확인

### 미검증

- 현재 실행 환경에 연결 가능한 내장 브라우저가 없어 실제 YouTube 곡 종료 후 다음 곡 이동은 확인하지 못했다.
- 실제 이어듣기 활성 상태에서 원본 다음 곡이 대기 플레이어에 준비되는지는 브라우저에서 확인하지 못했다.

## 위험 및 권장 다음 계획

- 셔플 목록의 시각적 다음 줄과 실제 다음 재생곡은 서로 다를 수 있다. 이는 “목록 표시만 랜덤, 재생은 원본 순서” 요구에 따른 의도된 동작이다.
- 실제 브라우저에서 셔플 활성 상태로 수동 다음곡, 자동 종료, 이어듣기 전환을 각각 2~3곡 확인하는 것이 좋다.
