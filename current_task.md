# 노동요 Cover Flow 확장 및 재생바 조절 작업 보고서

## 원래 요청

노동요 곡 정보는 앨범명을 제외하고 곡 제목과 아티스트만 표시한다. Cover Flow에는 이전 곡 2개와 다음 곡 2개를 보여주고, 이전·다음 곡으로 이동할 때 커버가 방향에 맞춰 움직이도록 한다. 기존 읽기 전용 재생바는 클릭과 드래그로 실제 재생 위치를 조절할 수 있게 변경한다.

## 완료한 작업

### 곡 정보

- 한 줄 정보에서 앨범명 요소와 관련 JavaScript 처리를 제거했다.
- `곡 제목 — 아티스트`만 한 줄로 표시한다.
- 아티스트 정보가 없으면 제목만 표시하고 구분자도 숨긴다.

### 5장 Cover Flow

- 표시 슬롯을 다음 다섯 개로 확장했다.
  1. 이전 두 번째 곡
  2. 이전 첫 번째 곡
  3. 현재 곡
  4. 다음 첫 번째 곡
  5. 다음 두 번째 곡
- 현재 곡은 중앙 전면, 인접 곡은 중간 깊이, 두 칸 떨어진 곡은 더 작고 흐린 뒤쪽 깊이로 표현했다.
- 순차 재생과 랜덤 재생 모두 기존 인접 곡 계산 함수를 사용한다.
- 곡 수가 부족하면 존재하지 않는 바깥 슬롯은 빈 커버로 처리한다.

### 커버 이동 애니메이션

- 다음 곡으로 이동하면 기존 다섯 커버가 왼쪽 슬롯으로 한 칸 이동한다.
- 이전 곡으로 이동하면 반대로 오른쪽 슬롯으로 한 칸 이동한다.
- 이동 중 중앙에 들어오는 커버는 크기, 밝기, 그림자가 함께 커진다.
- 바깥으로 나가는 커버는 이동하면서 투명해진다.
- 420ms의 짧은 전환을 사용하고 애니메이션 종료 뒤 새 다섯 커버를 슬롯에 정렬한다.
- 메인 버튼과 하단 리모컨의 이전·다음 버튼은 방향 힌트를 전달해 곡이 두 개뿐이어도 올바른 방향으로 애니메이션한다.
- 자동으로 다음 곡이 되는 경우에는 기존 Cover Flow 슬롯 관계로 방향을 판별한다.

### 조절 가능한 재생바

- 기존 표시용 `div` 재생바를 접근 가능한 `input type="range"`로 변경했다.
- 트랙 클릭, 마우스·터치 드래그, 키보드 방향키로 재생 위치를 선택할 수 있다.
- 선택한 비율을 현재 곡 길이로 환산해 기존 YouTube 플레이어의 `seekTo(seconds, true)`에 전달한다.
- 드래그 중 현재 시간과 진행 색상을 즉시 갱신한다.
- 플레이어 또는 곡 길이가 준비되지 않았으면 재생바를 비활성화한다.
- 현재 선택 곡과 플레이어의 실제 영상 ID가 다르면 잘못된 플레이어에 seek 명령을 보내지 않는다.
- 일반 플레이어와 이어듣기 활성 플레이어 모두 현재 `workMusicPlayer`만 조절한다.

## 기존 기능에 미치는 영향

- YouTube 플레이어 생성 옵션과 이벤트 핸들러는 변경하지 않았다.
- 이어듣기 전환, 볼륨, 실패 처리, 감시 타이머는 변경하지 않았다.
- 기존 이전·다음 곡 계산 결과는 그대로 사용하고 애니메이션 방향 정보만 추가했다.
- 재생바 조절은 사용자가 요청한 경우에만 `seekTo`를 호출하며 재생·일시정지 상태를 바꾸지 않는다.
- 기존 버튼 DOM 위치와 백그라운드 플레이어 구조를 유지했다.

## 변경 파일

- `index.html`: 5장 Cover Flow 구조, 앨범명 제거, 범위 입력 재생바로 변경
- `styles.css`: 두 단계 좌우 커버 깊이, 방향별 이동 애니메이션, 범위 입력 스타일 추가
- `src/features/workmusic.js`: 5곡 계산·애니메이션 상태 관리, 방향 판별, seek 처리 추가
- `current_task.md`: 이번 작업 보고서 작성

## 위험 및 권장 다음 계획

- 곡이 1~2개면 다섯 슬롯을 서로 다른 곡으로 채울 수 없어 일부 바깥 커버가 비게 된다.
- 매우 빠르게 이전·다음 버튼을 연속 입력하면 진행 중 애니메이션을 즉시 마무리하고 다음 이동을 시작한다.

- 일반 재생과 이어듣기 활성 상태에서 재생바의 클릭, 드래그, 방향키 조절을 각각 확인하는 것이 좋다.
