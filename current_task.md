# 노동요 곡별 재생선 및 페이드 핸들 구현 보고서

## 원래 요청

- 각 곡의 재생바 양끝에 흰색 동그라미를 표시한다.
- `< >` 핸들로 볼륨이 커지는 위치와 작아지는 위치를 곡마다 설정한다.
- 현재 재생 위치 손잡이는 빨간색으로 표시한다.
- 기존 요구처럼 핸들을 움직이면 YouTube도 해당 시간으로 즉시 이동한다.
- 현재 곡이 실제로 재생 중일 때만 핸들을 조작할 수 있게 한다.
- 구현 후 커밋하고 원격 저장소에 푸시한다.

## 완료한 작업

### 곡별 타임라인 데이터

각 곡 객체에 다음 값을 선택적으로 저장하도록 구현했다.

- `playbackStartSeconds`: 흰색 왼쪽 원의 곡 시작 시각
- `fadeInEndSeconds`: `<` 핸들의 페이드 인 종료 시각
- `fadeOutStartSeconds`: `>` 핸들의 페이드 아웃 시작 시각
- `playbackEndSeconds`: 흰색 오른쪽 원의 곡 종료 시각

기존 곡에 값이 없으면 시작은 0초, 종료는 곡 전체 길이를 사용한다. 페이드 지점은 이어듣기 설정 시간을 기본값으로 사용하며 곡 길이의 절반을 넘지 않게 제한한다.

### 재생바 UI

- 양끝에 흰색 원형 시작·종료 핸들을 추가했다.
- 재생선 위쪽에 `< >` 페이드 핸들을 추가했다.
- 현재 재생 위치의 range 손잡이와 이미 재생된 선을 빨간색으로 변경했다.
- 시작~페이드 인 종료와 페이드 아웃 시작~종료 구간을 붉은색 계열로 구분했다.
- 핸들은 현재 곡이 재생 중이고 플레이어 길이를 읽을 수 있을 때만 활성화된다.
- 핸들 제목에 해당 위치의 `분:초` 시간을 표시한다.

### 핸들 조작

- 네 핸들 모두 Pointer Events 기반으로 드래그할 수 있다.
- 드래그 중 해당 핸들의 시각으로 YouTube 플레이어를 즉시 이동한다.
- 시작 원을 옮기면 기존 페이드 인 길이를 유지하며 `<` 핸들도 함께 이동한다.
- 종료 원을 옮기면 기존 페이드 아웃 길이를 유지하며 `>` 핸들도 함께 이동한다.
- `< >` 핸들은 시작·종료 원과 서로 교차하지 않도록 제한한다.
- 드래그가 끝날 때만 클라우드 저장을 요청해 저장 호출이 과도하게 발생하지 않게 했다.

### 실제 재생 및 볼륨

- 곡을 새로 재생하면 저장된 시작 지점으로 이동한다.
- 일반 재생에서는 100ms 간격으로 현재 시각을 확인해 곡별 페이드 인·아웃 볼륨을 적용한다.
- 저장된 종료 지점에 도달하면 현재 표시 목록의 다음 곡으로 이동한다.
- 이어듣기에서는 현재 곡의 `>` 지점부터 이전 곡을 줄이고 다음 곡의 시작~`<` 길이에 맞춰 다음 곡을 키운다.
- 이어듣기 대기 플레이어도 다음 곡의 저장된 시작 지점에 맞춰 준비한다.

### 안전장치

- 핸들 조작과 볼륨 감시는 현재 플레이어의 실제 YouTube 영상 ID가 현재 곡의 영상 ID와 정확히 일치할 때만 실행한다.
- 이전 플레이어나 대기 플레이어의 늦은 이벤트가 다른 곡의 설정·볼륨에 적용되지 않게 했다.
- 기존 플레이어 생성, 오류 처리, 셔플 표시 순서와 실패 기록 구조는 변경하지 않았다.

- `index.html`: 흰색 시작·종료 원과 `< >` 핸들 HTML 추가

- `src/features/workmusic.js`: 곡별 값 계산·드래그·저장·seek·볼륨 페이드·종료 이동 구현

## 검증 결과

### 완료

- `node --check src/features/workmusic.js` 통과
- `npm run lint` 통과
- `npm run format:check` 통과
- `git diff --check` 통과
- 로컬 HTTP 서버에서 네 타임라인 핸들 HTML이 제공되는 것을 확인
- Firebase 노동요 저장이 곡 객체를 펼쳐 저장하므로 새 곡별 필드가 보존되는 것을 확인
- 일반 플레이어와 이어듣기 플레이어가 각각 저장된 시작·페이드·종료 지점을 참조하는 코드 경로 확인
- 현재 플레이어 영상 ID 검증이 핸들 조작과 볼륨 감시 앞에 있는지 확인

### 미검증

- 작업 세션에 앱 내 브라우저가 제공되지 않아 실제 마우스 드래그 모양과 YouTube seek 동작은 직접 검증하지 못했다.
- 실제 음원으로 일반 재생 페이드와 이어듣기 교차 페이드의 음량 곡선을 청취 검증하지 못했다.
- YouTube 자동재생 정책과 네트워크 지연이 있는 환경의 전환은 배포 후 확인이 필요하다.
