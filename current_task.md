# 노동요 일시정지 시 목록 위치 유지 작업 보고서

## 원래 요청

노래 재생 중 재생 버튼을 눌러 곡을 멈추면 노동요 목록이 맨 위로 돌아가는 문제를 수정한다.

## 확인된 원인

- 일시정지는 현재 곡과 곡 목록이 바뀌지 않는 동작인데도 `toggleWorkMusicPlay()`가 마지막에 `renderWorkMusic()`을 호출하고 있었다.
- 일반 YouTube 플레이어의 `PAUSED` 이벤트와 이어듣기 플레이어의 `PAUSED` 이벤트도 각각 `renderWorkMusic()`을 다시 호출했다.
- `renderWorkMusic()`은 `workMusicList.innerHTML = ''`로 목록 전체 DOM을 삭제한 뒤 모든 곡 행을 다시 생성한다.
- 목록이 삭제되는 순간 문서 높이가 줄어 브라우저가 현재 페이지 스크롤을 위쪽으로 보정하고, 목록이 다시 생겨도 기존 위치로 돌아가지 않았다.
- 결과적으로 현재 곡 인덱스는 유지되지만 사용자가 보고 있던 목록 위치가 맨 위쪽으로 이동했다.

## 완료한 작업

### 재생 상태 전용 갱신 분리

- `renderWorkMusicPlaybackState()`를 추가했다.
- 재생·일시정지 상태에서는 전체 목록을 다시 만들지 않고 다음 항목만 갱신한다.
  - 메인 재생 버튼 아이콘과 접근성 설명
  - 하단 리모컨 재생 상태와 곡 정보
  - Cover Flow와 곡 정보·진행 시간
  - 목록 현재 곡 행의 활성 상태와 재생·일시정지 아이콘

### 목록 행 부분 갱신

- 각 `.workmusic-item`에 원본 곡 인덱스를 `data-index`로 저장했다.
- `renderWorkMusicListPlaybackState()`가 기존 목록 DOM을 유지한 채 현재 행만 찾는다.
- 현재 곡 행에는 `active` 클래스를 유지하고 재생 상태에 맞는 아이콘만 교체한다.
- 다른 곡 행은 재생 아이콘 상태로 유지한다.

### 일반·이어듣기 경로 통일

- 메인 재생 버튼과 하단 리모컨의 재생·일시정지 동작에서 부분 갱신을 사용한다.
- 일반 YouTube 플레이어의 `PLAYING`, `PAUSED`, `ENDED` 상태에서 부분 갱신을 사용한다.
- 이어듣기 플레이어의 `PLAYING`, `PAUSED` 상태에서도 같은 부분 갱신을 사용한다.
- 곡 인덱스가 실제로 바뀌는 다음곡·이전곡·자동 전환은 기존 전체 렌더를 유지한다.
- 재생 성공으로 기존 실패 표시를 삭제해야 하는 특수 상황에만 전체 목록을 다시 그려 오류 배지가 정확히 제거되게 했다.

## 기존 기능에 미치는 영향

- 재생·일시정지 명령과 YouTube 플레이어 제어 방식은 변경하지 않았다.
- 현재 곡 인덱스, 셔플 표시, 원본 순서 재생, 이어듣기 순서는 변경하지 않았다.
- 곡 클릭, 다음곡·이전곡, 재생바, 볼륨, 실패 처리 기능은 유지된다.
- 목록 DOM을 유지하므로 기존 행 클릭 및 설정 버튼 이벤트도 그대로 유지된다.
