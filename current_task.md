# 현재 작업 보고서

## 원래 요청

- 처음 접속하면 노동요 목록은 저장된 원래 순서로 표시한다.
- 셔플 버튼을 누르면 목록 자체를 무작위 순서로 바꾼다.
- 재생은 별도의 무작위 선택 없이 화면에 표시된 목록 순서대로 진행한다.
- 재생 중 셔플을 켜면 현재 듣는 곡을 목록 첫 번째에 두고 나머지 곡만 섞는다.
- 구현 후 커밋하고 원격 저장소에 푸시한다.

## 완료한 작업

### 목록 순서와 재생 순서 통합

- 셔플된 화면 목록 순서를 일반 YouTube 재생목록의 순서로 그대로 사용하도록 변경했다.
- 이어듣기 활성 플레이어와 대기 플레이어도 같은 순서에서 다음 곡을 찾도록 변경했다.
- 이전/다음 버튼과 커버플로우의 이전곡·다음곡 계산도 화면 목록 순서를 사용하도록 변경했다.
- 재생 실패 후 다음 곡을 찾는 기존 코드도 갱신된 현재 재생 순서를 그대로 사용한다.

### 셔플 동작 변경

- 재생 중 셔플을 켜면 현재 곡의 내부 인덱스를 첫 번째에 고정한 뒤 나머지만 무작위로 섞는다.
- 정지 상태에서 셔플을 켜면 전체 목록을 섞고, 섞인 첫 곡을 현재 곡으로 맞춘다.
- 셔플을 끄면 원래 저장 순서로 돌아간다.
- 셔플로 플레이어 목록을 다시 만들 때 기존 재생 시간을 읽어 새 플레이어가 준비된 뒤 같은 시간으로 이동하도록 했다.

### 최초 접속 순서

- 저장 데이터에 과거 `random` 값이 남아 있어도 새 접속 또는 저장 데이터 불러오기 시 `sequential`로 시작한다.
- 메모리에 남은 셔플 캐시와 재생 순서도 초기화한다.
