# 노동요 Cover Flow 좌우 여백 및 인접 커버 이동 작업 보고서

## 원래 요청

Cover Flow 커버가 좌우 끝에 완전히 붙어 어색하므로 좌우에 약 10px 여백을 만든다. 그 상태에서 이전곡과 다음곡 커버만 각각 10px씩 현재곡 커버 방향으로 이동한다.

## 완료한 작업

### 좌우 여백

- Cover Flow가 사용하는 `.workmusic-player-view` 너비를 `100%`에서 `calc(100% - 20px)`로 변경했다.
- 뷰를 가운데 정렬해 좌우에 각각 10px의 구조적인 여백을 만들었다.
- 커버 크기와 위치에 사용하는 컨테이너 단위도 줄어든 가용 폭을 기준으로 계산된다.
- 바깥 커버의 기존 크기와 위치 변수는 변경하지 않았다.

### 이전곡·다음곡 중앙 이동

- 인접 커버 중심 위치를 `calc(min(185px, 24.42cqi) - 10px)`로 변경했다.
- 이전곡은 오른쪽으로 10px, 다음곡은 왼쪽으로 10px 이동한다.
- 전전곡과 다다음곡에는 추가 10px 이동을 적용하지 않았다.
- 다음·이전 애니메이션의 인접 슬롯도 같은 변수를 사용하므로 이동 중에도 변경된 위치로 자연스럽게 연결된다.

## 수치 검증

- 800px, 600px, 360px 컨테이너에서 구조적인 좌우 여백이 각각 10px인지 확인했다.
- 같은 세 폭에서 인접 커버에 추가 적용되는 중앙 방향 이동량이 정확히 10px인지 확인했다.
- 커버 자체의 기존 배치 여유를 포함한 실제 바깥 커버와 화면 끝의 간격은 폭에 따라 약 11.7px 이상이다.

## 기존 기능에 미치는 영향

- CSS의 뷰 너비와 인접 커버 위치 변수만 변경했다.
- 커버 크기 단계, 바깥 커버 위치, 평면 좌우 이동, 검정 마스크는 변경하지 않았다.
- HTML과 JavaScript는 변경하지 않았다.
- YouTube 재생, 이어듣기, 이전·다음 곡 선택, 재생바 조절 로직에는 영향이 없다.
- 직전의 미커밋 UI 작업도 그대로 유지했다.
